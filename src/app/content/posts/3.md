# Deploying Plausible Analytics on a Reusable Terraform Environment

In my previous article, I set up Plausible Analytics using a series of CLI commands. While that approach works, it's not ideal for repeatability or collaboration. In this guide, we'll convert that setup into a fully modular, reusable Terraform project, capable of deploying Plausible into different environments like dev, staging, and prod. We'll also use the Cloudflare Terraform provider to update DNS records automatically.

---

## ✨ Why Use Terraform?

- **Consistency**: Infrastructure as Code reduces human error.
- **Modularity**: Easy to reuse and maintain.
- **Environment Separation**: Manage dev/prod configs cleanly.
- **Automation**: Integrate with CI/CD for deployments.

---

## 📂 Project Structure

```bash
terraform/
├── main.tf
├── variables.tf
├── outputs.tf
├── environments/
│   ├── dev/
│   │   └── terraform.tfvars
│   └── prod/
│       └── terraform.tfvars
├── modules/
│   ├── aks/
│   ├── plausible/
│   └── dns/
```

Each module encapsulates a responsibility:

- `aks`: Deploys an AKS cluster
- `plausible`: Installs Plausible and its dependencies via Helm
- `dns`: Updates Cloudflare DNS records

---

## ⚙️ Setting Up the AKS Module

Inside `modules/aks/main.tf`:

```hcl
resource "azurerm_kubernetes_cluster" "main" {
  name                = var.name
  location            = var.location
  resource_group_name = var.resource_group_name

  default_node_pool {
    name       = "default"
    node_count = 2
    vm_size    = "Standard_DS2_v2"
  }

  identity {
    type = "SystemAssigned"
  }
}
```

Outputs can include the kubeconfig:

```hcl
output "kube_config" {
  value     = azurerm_kubernetes_cluster.main.kube_config_raw
  sensitive = true
}
```

---

## 💰 Deploying Plausible via Helm

In `modules/plausible/main.tf`:

```hcl
resource "helm_release" "plausible" {
  name             = "plausible"
  repository       = "https://charts.plausible.io"
  chart            = "plausible"
  namespace        = "analytics"
  create_namespace = true

  values = [
    file("${path.module}/values.yaml")
  ]
}
```

Include ClickHouse, PostgreSQL, and Ingress values in `values.yaml`. You can store secrets securely using Key Vault or pull them from environment variables in CI.

---

## 🔍 Setting Up DNS with Cloudflare

In `modules/dns/main.tf`:

```hcl
provider "cloudflare" {
  api_token = var.cloudflare_api_token
}

resource "cloudflare_record" "plausible" {
  zone_id = var.cloudflare_zone_id
  name    = "plausible"
  value   = var.ingress_ip
  type    = "A"
  ttl     = 300
  proxied = true
}
```

The `ingress_ip` can be passed from the output of the NGINX ingress controller service.

---

## ⚡ Environment Configuration

Create files like `environments/dev/terraform.tfvars`:

```hcl
name                  = "dev-cluster"
location              = "westeurope"
resource_group_name   = "analytics-rg"
cloudflare_api_token  = "your-token"
cloudflare_zone_id    = "your-zone-id"
ingress_ip            = "123.123.123.123"
```

---

## ▶️ Running Terraform

```bash
cd terraform
terraform init
terraform workspace new dev
terraform workspace select dev
terraform apply -var-file=environments/dev/terraform.tfvars
```

Terraform will:

- Deploy the AKS cluster
- Install Plausible via Helm
- Update Cloudflare DNS
